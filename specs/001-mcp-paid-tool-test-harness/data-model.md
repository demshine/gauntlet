# Data Model

## Policy

Defines deterministic spending boundaries for one agent.

Fields:

- `policy_id`: stable policy identifier
- `version`: policy schema version
- `agent_id`: agent covered by policy
- `max_amount_per_payment`: maximum single payment amount
- `max_total_budget`: maximum cumulative task budget
- `budget_window`: V0.1 supports `task`
- `currency`: required fiat or token currency
- `chain`: optional network constraint
- `allowed_merchants`: exact merchant IDs allowed
- `blocked_merchants`: exact merchant IDs denied
- `requires_review_above`: amount threshold for review
- `expires_at`: policy expiry timestamp
- `max_payments_per_task`: task-level frequency limit
- `max_quote_drift_percentage`: allowed quote drift
- `required_metadata`: required request metadata keys

## Merchant

Represents fixture-defined merchant identity. V0.1 does deterministic matching only.

Fields:

- `merchant_id`
- `canonical_merchant_id`
- `merchant_type`
- `merchant_domain`
- `merchant_wallet_address`
- `merchant_provider_id`
- `merchant_display_name`

## Quote

Represents a price quote returned before the agent creates a payment request.

Fields:

- `quote_id`
- `merchant_id`
- `quoted_amount`
- `quoted_currency`
- `quoted_at`
- `expires_at`
- `item_description`
- `final_amount`
- `drift_amount`
- `drift_percentage`

## PaymentRequest

Represents one simulated payment request generated by an agent.

Fields:

- `request_id`
- `task_id`
- `session_id`
- `idempotency_key`
- `original_request_id`
- `retry_count`
- `previous_decision_id`
- `provider_request_id`
- `provider_transaction_reference`
- `merchant_order_id`
- `payment_session_id`
- `quote_id`
- `price_quote_expires_at`
- `agent_id`
- `merchant`
- `amount`
- `currency`
- `token`
- `chain`
- `purpose`
- `tool_source`
- `provider`
- `created_at`
- `metadata`

## History

Represents explicit state for rules that cannot be evaluated from one request.

Fields:

- `task_id`
- `session_id`
- `prior_requests`
- `prior_decisions`
- `prior_receipts`
- `total_amount_spent`
- `payment_count_for_task`
- `used_idempotency_keys`

## Scenario

Represents a runnable failure case.

Fields:

- `scenario_id`
- `title`
- `description`
- `category`
- `risk_represented`
- `policy`
- `merchant`
- `quote`
- `payment_request`
- `history`
- `expected_decision`
- `expected_reason_codes`
- `advisory_expectations`

## Receipt

Represents the structured evaluation result.

Fields:

- `receipt_id`
- `trace_id`
- `scenario_id`
- `timestamp`
- `policy_snapshot`
- `merchant_snapshot`
- `quote_snapshot`
- `payment_request_snapshot`
- `decision`
- `reason_codes`
- `advisory_warnings`
- `evaluation_trace`
- `redaction_policy`
- `redacted_fields`
- `simulator_version`

## EvaluationTrace

Represents rule-by-rule reasoning.

Fields:

- `evaluated_rules`
- `rule_id`
- `rule_version`
- `input_fields_used`
- `operator`
- `expected_value`
- `actual_value`
- `result`
- `severity`
- `decision_contribution`
